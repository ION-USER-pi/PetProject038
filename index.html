Web Services Modeling Ontology

WSMO provides a conceptual framework and a formal language for semantically describing all relevant aspects of web services to facilitate the automation of discovering, combining and invoking electronic services over the web.

Discover WSMO Resources

What is WSMO?

The Web Service Modeling Ontology (WSMO) is a conceptual model for describing various aspects related to Semantic Web Services. WSMO is based on the Web Service Modeling Framework (WSMF) and identifies four main elements for describing semantic web services:

Ontologies

Provide the terminology used by other WSMO elements to describe the relevant aspects of the domains of discourse. They define a formally specified terminology of the information used by all other components.

Web Services

Describe the functional and behavioral aspects of a Web service. This includes the capability of the service, its interfaces, and internal working mechanisms.

Goals

Define the user's objectives when consulting a Web service. Represent the user desires with respect to the requested functionality and specify what a service requester is looking for.

üîÑ

Mediators

Describe elements that handle interoperability problems between different WSMO elements. They resolve heterogeneity issues that arise when connecting different systems.

WSMO Architecture

The architecture of WSMO follows a layered approach, each layer building on top of the previous one to provide a comprehensive framework for semantic web services.

Conceptual Model

The top-level elements of Ontologies, Web Services, Goals, and Mediators form the conceptual backbone of WSMO.

Formal Language: WSML

Web Service Modeling Language (WSML) provides a formal syntax and semantics for the WSMO conceptual model.

Execution Environment: WSMX

Web Service Modeling Execution Environment (WSMX) serves as a reference implementation for WSMO, providing runtime capabilities.

Real-world applications of WSMO in domains like e-commerce, enterprise application integration, and more.

Core Principles of WSMO

WSMO is built upon several foundational principles that guide its design and application:

Web Compliance +
WSMO inherits and extends the concept of URI (Uniform Resource Identifier) for unique identification of resources. Furthermore, WSMO adopts the concept of namespaces for denoting consistent information spaces, and supports XML and other W3C Web technology recommendations.
Ontology-Based +
Ontologies are used as the data model throughout WSMO. All resource descriptions and all data interchanged during service usage are based on ontologies. Ontological definitions allow for reuse, inheritance, and well-defined semantics.
Strict Decoupling +
WSMO follows the principle of strict decoupling of various components that realize a service-oriented application. This means that each resource is described independently, without regard to possible usage or interactions with other resources.
Centrality of Mediation +
Mediation addresses the heterogeneity that naturally exists in a distributed environment. WSMO recognizes the importance of mediation for resolving heterogeneities by including mediators as top-level elements to connect different WSMO elements.
Separation of Service Interface and Protocol +
WSMO differentiates between the capabilities of a service (what a service can do) and the service interface (how the functionality can be achieved). This separation allows for flexible binding and improved reusability.
Description versus Implementation +
WSMO differentiates between the description of Semantic Web Services elements and their actual implementation. Keeping the description separate from the implementation allows for more flexibility in deployment and usage.
